# BuildAI Framework - 项目规格说明书 (SPEC)

## 1. 项目概述

### 1.1 项目定位
构建一个 **B2B SaaS 中控平台**，允许管理员通过统一的 Web 界面：
- 管理多用户和多服务器
- 配置 AI API Keys（用户级 + 系统级）
- 一键部署应用到远程服务器
- 实时监控服务器状态和应用运行状态

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **SaaS 中控** | 主控平台，提供 Web UI 和 API |
| **Agent** | 部署在远程服务器上的服务端程序，接收中控指令 |
| **应用 (App)** | 可独立运行的功能模块，如 ServerManager、PolymarketBot |
| **用户** | SaaS 平台的租户，拥有独立的 API Key 和资源配置 |

### 1.3 架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                      SaaS 中控平台 (Controller)                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐  │
│  │  Web UI     │  │  API Server │  │  Task Scheduler         │  │
│  │  (React)    │  │  (Go/Node)  │  │  (Job Queue)            │  │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │ HTTPS + mTLS
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Agent (部署在各服务器)                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐  │
│  │  HTTP API   │  │  App Runner │  │  Monitoring             │  │
│  │  Server     │  │  (Docker)   │  │  (Metrics/Logs)         │  │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 功能需求

### 2.1 中控平台 (SaaS Controller)

#### 2.1.1 用户管理
- [ ] 用户注册/登录（邮箱 + 密码 / OAuth）
- [ ] 用户角色：Admin / Manager / User
- [ ] 用户级 API Key 生成与管理

#### 2.1.2 服务器管理
- [ ] 服务器注册（IP、SSH 密钥、标签）
- [ ] 服务器健康检查（心跳检测）
- [ ] 服务器资源监控（CPU、内存、磁盘）
- [ ] 批量命令执行

#### 2.1.3 API Key 管理
- [ ] 系统级大模型 API Key 配置（OpenAI、Claude、Kimi 等）
- [ ] 用户级 API Key 分配
- [ ] API Key 使用统计和限额

#### 2.1.4 应用市场
- [ ] 应用列表展示
- [ ] 应用一键部署到指定服务器
- [ ] 应用配置管理（环境变量、端口映射）
- [ ] 应用启动/停止/重启

#### 2.1.5 监控告警
- [ ] 服务器离线告警
- [ ] 应用异常告警
- [ ] 资源使用阈值告警

### 2.2 Agent 服务端

#### 2.2.1 核心功能
- [ ] 向中控注册并维持心跳
- [ ] 接收并执行中控指令（Docker 操作、命令执行）
- [ ] 上报服务器状态和应用状态
- [ ] 日志收集和转发

#### 2.2.2 安全配置
- [ ] mTLS 双向认证
- [ ] 指令签名验证
- [ ] 敏感操作二次确认

### 2.3 应用：远程服务器管理 (ServerManager)

#### 2.3.1 功能
- [ ] SSH 终端（Web Terminal）
- [ ] 文件管理器（上传/下载/编辑）
- [ ] 进程管理（查看/杀死进程）
- [ ] 服务管理（systemd 服务操作）
- [ ] 日志查看器

### 2.4 应用：Polymarket 交易机器人 (PolymarketBot)

#### 2.4.1 功能
- [ ] Polymarket API 对接
- [ ] 市场数据抓取
- [ ] 交易策略执行
- [ ] 持仓管理和盈亏统计
- [ ] 风控规则（止损/止盈）

## 3. 技术栈

### 3.1 中控平台
| 层级 | 技术 |
|------|------|
| 前端 | React + Ant Design / Element Plus |
| 后端 | Node.js (Express/NestJS) 或 Go (Gin) |
| 数据库 | PostgreSQL + Redis |
| 消息队列 | Redis / RabbitMQ |
| 任务调度 | Bull / Agenda |

### 3.2 Agent
| 功能 | 技术 |
|------|------|
| 服务 | Go / Python |
| 容器 | Docker + Docker Compose |
| 通信 | HTTP/2 + gRPC（可选） |

### 3.3 应用
| 应用 | 技术 |
|------|------|
| ServerManager | Node.js + WebSocket (xterm.js) |
| PolymarketBot | Python + Web3.py |

## 4. 安全设计

### 4.1 通信安全
- 中控 ↔ Agent：mTLS + AES-256-GCM 加密
- 用户 ↔ 中控：HTTPS + JWT
- API Key：bcrypt 存储，使用 HMAC 签名

### 4.2 访问控制
- RBAC 权限模型
- 操作审计日志
- 敏感操作确认机制
