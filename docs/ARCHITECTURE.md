# BuildAI Framework - 系统架构文档

## 1. 系统概述

BuildAI Framework 是一个 B2B SaaS 开发框架，提供完整的多租户应用开发、部署和管理能力。框架采用分层架构设计，包含核心框架层、应用层和基础设施层。

### 2.2 分层说明

| 层级 | 职责 | 组件 |
|------|------|------|
| 接入层 | 流量接入、负载均衡、静态资源 | Nginx、CDN |
| 控制层 | 业务逻辑、权限控制、任务调度 | Controller Service |
| 数据层 | 数据持久化、缓存、消息队列 | PostgreSQL、Redis、MinIO |
| 代理层 | 远程服务器管理、应用执行 | Agent Service |
| 应用层 | 具体业务应用 | 各应用容器 |

### 5.2 多租户数据隔离策略

BuildAI Framework 采用 **共享数据库、独立 Schema** 的多租户策略：

```
PostgreSQL Database: buildai
├── Schema: tenant_abc123  (租户 abc123 的数据)
│   ├── products
│   ├── orders
│   └── customers
├── Schema: tenant_def456  (租户 def456 的数据)
│   ├── products
│   ├── orders
│   └── customers
└── Schema: public  (系统共享数据)
    ├── users
    ├── tenants
    └── api_keys
```

**优点：**
- 数据完全隔离，安全性高
- 支持租户级备份和恢复
- 便于租户数据迁移

## 6. 通信协议

### 6.1 中控与 Agent 通信

```
┌─────────────┐                    ┌─────────────┐
│  Controller │◄────── mTLS ──────►│    Agent    │
│  (中控)     │    + HTTP/2        │  (代理)     │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       │ 1. 注册/心跳                      │ 1. 启动注册
       │ 2. 指令下发                       │ 2. 执行指令
       │ 3. 状态上报                       │ 3. 上报状态
       │ 4. 日志转发                       │ 4. 收集日志
```

### 6.2 消息格式

```protobuf
// Command 指令
message Command {
  string id = 1;
  string type = 2;      // docker, exec, file, config
  bytes payload = 3;
  string signature = 4;
  int64 timestamp = 5;
}

// Response 响应
message Response {
  string command_id = 1;
  bool success = 2;
  bytes data = 3;
  string error = 4;
}

// Heartbeat 心跳
message Heartbeat {
  string agent_id = 1;
  int64 timestamp = 2;
  SystemMetrics metrics = 3;
  repeated ContainerStatus containers = 4;
}
```

```
│  (服务间通信)                            │
├─────────────────────────────────────────┤
│  Layer 3: JWT Token                     │
│  (用户认证)                              │
├─────────────────────────────────────────┤
│  Layer 2: mTLS 双向证书                  │
│  (中控-代理通信)                         │
├─────────────────────────────────────────┤
│  Layer 1: HTTPS/TLS                     │
│  (传输加密)                              │
└─────────────────────────────────────────┘
```

### 7.2 数据安全

- **数据库**: 敏感字段加密存储（AES-256）
- **文件存储**: 服务端加密（SSE-S3）
- **密码**: bcrypt 哈希
- **API Key**: 哈希存储，仅显示一次

## 9. 监控与告警

### 9.1 监控指标

| 层级 | 指标 | 采集方式 |
|------|------|----------|
| 基础设施 | CPU、内存、磁盘、网络 | Agent 采集 |
| 应用 | QPS、延迟、错误率 | Middleware |
| 业务 | 用户数、订单数、收入 | 业务埋点 |

### 9.2 告警规则

```yaml
alerts:
  - name: high_cpu
    condition: cpu_usage > 80%
    duration: 5m
    severity: warning
    
  - name: service_down
    condition: http_status != 200
    duration: 1m
    severity: critical
    
  - name: disk_full
    condition: disk_usage > 90%
    duration: 0s
    severity: critical
```

## 10. 技术选型对比

| 组件 | 选型 | 备选 | 原因 |
|------|------|------|------|
| 后端框架 | NestJS | Express | 模块化、TypeScript 友好 |
| 前端框架 | React | Vue | 生态丰富 |
| 数据库 | PostgreSQL | MySQL | JSON 支持、扩展性强 |
| 缓存 | Redis | Memcached | 支持更多数据结构 |
| 消息队列 | Redis | RabbitMQ | 简化部署 |
| 对象存储 | MinIO | AWS S3 | 私有化部署 |
| 日志分析 | ClickHouse | ELK | 高性能、低成本 |
